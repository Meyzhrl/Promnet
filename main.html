Summary

Git

Git adalah alat yang digunakan untuk mengelola source code suatu program dengan cara menentukan baris serta kode-kode yang perlu diganti atau ditambahkan. Ini juga berfungsi sebagai control system untuk menjalankan proyek pengembangan software.

Kelebihan Git

  1. Mudah bagi developer untuk bekerja sama dalam pengembangan project.
  2. Membantu dalam dokumentasi project, baik dari sisi source code ataupun perubahan fungsi.
  3. Setiap perubahan pada file di-commit, sehingga dapat terlihat.
  4. Repositori dapat disimpan secara offline maupun online.
  5. Mudah dipahami dalam desainnya.

Perintah Dasar Git

  1. Git Init: Untuk membuat repository lokal.
  2. Git Status: Untuk mengecek status repository lokal.
  3. Git Add: Untuk menambahkan file baru ke repository yang baru dipilih.
  4. Git Commit: Untuk menyimpan perubahan pada repository lokal.
  5. Git Push: Untuk mentransfer perubahan file ke repository jarak jauh setelah melakukan perubahan.
  6. Git Checkout: Digunakan untuk menukar cabang aktif dengan cabang yang dipilih.
  7. Git Merge: Untuk menggabungkan cabang aktif dengan cabang yang dipilih.
  8. Git Clone: Untuk membuat salinan repository lokal.
  9. Git Pull: Mengambil perubahan terbaru dari repositori jarak jauh dan menggabungkannya dengan cabang lokal.
  10. Git Reset: Digunakan untuk mengubah status dan sejarah commit dalam repositori.
